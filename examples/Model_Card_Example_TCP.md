# Model Card: TCP-сервер с epoll (проект Иванова И., группа 123)

## 1. Объект моделирования
- **Система:** многопоточный TCP-сервер, обрабатывающий короткие текстовые запросы.
- **Язык:** C++17.
- **Требования:** RFC 793 (TCP), доп. условия: таймаут на чтение заголовка 2 с, на тело 3 с.

## 2. Поведение
- Вход: соединение, затем сообщение вида `HEADER\nBODY\n`.
- Выход: ответ `RESPONSE\n` или закрытие соединения по таймауту.
- Ограничения: длина сообщения ≤ 4096 байт, макс. число соединений 20000 (C++).

## 3. Допущения
- Сеть не теряет пакеты и не дублирует их.
- Все клиенты отправляют корректные запросы.
- Время обработки запроса одинаково для всех сообщений (80 мкс).

## 4. Формальная модель
- **Тип:** конечный автомат с состояниями: WAIT_HEADER, READ_BODY, PROCESS, WRITE_RESP.
- **Параметры:** вероятности переходов зависят от таймаутов.
- **Схема:** (приложена отдельно)

## 5. Валидация
- Написаны 5 юнит-тестов на каждый таймаут.
- Трассы собраны с 10 000 соединений, отклонение предсказания модели от факта <5%.

## 6. Границы применимости
- Модель точна при нагрузке до 50 000 req/s.
- Не учитывает влияние фрагментации памяти.
- Работает только на Linux с epoll.
